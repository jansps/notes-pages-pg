# notes-pages-pg

用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库用于存放和管理一系列独立的 HTML 页面。这些页面彼此独立，不依赖于特定域名或网站环境，适合作为静态内容归档或临时页面发布使用。仓库本身不包含动态后端，所有页面均为纯前端 HTML 文件，可直接通过浏览器访问。

## 目录说明

```
notes-pages-pg/
├── pages/          # 存放所有归档的 HTML 页面
├── assets/         # 页面引用的公共资源（CSS、JS、图片等）
└── README.md       # 本文件
```

- **pages/**：每个 HTML 页面为一个独立文件，命名尽量简洁，便于识别。
- **assets/**：存放页面可能用到的公共资源文件，如样式表、脚本、图像等。资源按需组织，避免冗余。

## 页面归档说明

- 所有页面均为静态 HTML，不依赖外部服务或数据库。
- 页面内容以归档为主，不保证长期更新或维护。
- 页面可能包含简单的交互功能（如 JavaScript 实现的表单、动画等），但均不涉及用户数据收集或网络请求。
- 如需新增页面，可直接在 `pages/` 目录下添加 HTML 文件，并在 `assets/` 中补充所需资源。

## 维护说明

- 本仓库由个人维护，更新频率不固定。
- 欢迎提交 Issue 或 Pull Request，但需确保所提交内容为独立、无依赖的 HTML 页面。
- 不建议在此仓库中存放敏感信息、商业内容或违反社区准则的材料。
- 仓库可能不定期清理或重组目录结构，届时会更新 README 说明。
